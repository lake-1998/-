## 一、全局约定

### 1.1 页面 ID 命名规则

| 页面ID | 页面名称 | 路由路径（示意）|
|-------|---------|--------------|
| P01 | 首页 | `/pages/index/index` |
| P02 | 搜索结果页 | `/pages/search/result` |
| P03 | 网点详情页 | `/pages/store/detail` |
| P04A | 下单页（搜索路径）| `/pages/order/create` |
| P04B | 下单页（扫码路径）| `/pages/order/scan-create` |
| P05 | 支付成功页 | `/pages/order/success` |
| P06 | 订单列表页 | `/pages/order/list` |
| P07 | 订单详情页 | `/pages/order/detail` |
| P08 | 「我的」页面 | `/pages/user/index` |

### 1.2 状态变量命名约定

文档中使用以下变量描述条件：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `IS_LOGIN` | Boolean | 用户是否已登录（绑定手机号）|
| `HAS_ACTIVE_ORDER` | Boolean | 用户当前是否有进行中的订单 |
| `ORDER_STATUS` | Enum | 当前订单状态（见§2.2）|
| `STORE_TYPE` | Enum | 网点类型：`LOCKER`（智能柜）/`MANUAL`（人工寄存）|
| `CITY_OPEN` | Boolean | 当前城市是否开通某功能（如行李配送）|
| `box_id` | String | URL参数，有值=扫码进入，无值=搜索路径进入 |

### 1.3 交互动效规范

| 场景 | 动效类型 | 时长 |
|------|---------|------|
| 页面前进 | 右滑入 | 300ms |
| 页面返回 | 左滑出 | 300ms |
| 底部弹层出现 | 上推 | 250ms |
| 底部弹层消失 | 下收 | 200ms |
| 地图选中网点 | 卡片弹性浮起 | 200ms + 弹性回弹 |
| Toast 提示 | 淡入淡出 | 显示2秒后自动消失 |
| 加载状态 | 骨架屏（非转圈） | — |

## 二、全局状态定义

### 2.1 用户登录状态

```
用户登录状态机：

  [未授权] ──── 点击需要登录的操作 ───→ [授权弹窗]
      ↑                                      │
      │                                 用户授权手机号
      │                                      ↓
   退出登录 ←──────────────────────── [已登录]
```

| 状态 | 定义 | 影响范围 |
|------|------|---------|
| 未登录 | 未完成手机号授权 | 下单功能不可用；历史搜索依赖localStorage（可用）|
| 已登录 | 已完成手机号授权 | 全功能可用；下单、查看订单等需登录操作解锁 |

**登录触发时机**：
- ✅ 点击"立即预订"按钮时触发（非更早）
- ✅ 点击"查看订单"时触发
- ❌ 不在首页/详情页进入时强制触发
- ❌ 不在查看地址/电话时触发（下单才解锁，非登录解锁）

### 2.2 订单状态枚举

| ORDER_STATUS 值 | 用户端显示文案 | 说明 |
|----------------|-------------|------|
| `PENDING` | 等待寄存 | 已支付，待到店 |
| `IN_PROGRESS` | 寄存中 | 已到店开始寄存 |
| `REFUND_PENDING` | 退款处理中 | 用户申请取消，平台处理中 |
| `OVERDUE` | ⚠️ 已超时，请尽快续费 | 超过预约结束时间未取回 |
| `COMPLETED` | 已完成 | 正常取回 |
| `CANCELLED` | 已取消 | 取消成功 |

### 2.3 信息解锁状态

| 信息类型 | 未下单 | 已下单 |
|---------|-------|-------|
| 网点位置（地图精确点）| ✅ 可见 | ✅ 可见 |
| 导航能力 | 🔒 锁定 | ✅ 解锁 |
| 精确地址文字 | 🔒 仅显示区域描述 | ✅ 精确地址 |
| 商家电话 | 🔒 锁定 | ✅ 解锁 |
| 实景大图 | 🔒 模糊遮罩 | ✅ 清晰显示 |
| 精确店铺名称 | 🔒 模糊描述 | ✅ 精确名称 |

## 三、全局跳转架构

```
小程序入口
    │
    ├── [扫码进入] box_id 参数存在 ──────────────────────→ P04B 扫码下单页
    │                                                           │
    └── [其他入口] 无 box_id ───────────────────────────→ P01 首页
                                                               │
                        ┌──────────────────────────────────────┤
                        │                                      │
                   搜索/点击附近网点                       底部Tab导航
                        │                                      │
                        ↓                              ┌───────┴────────┐
                   P02 搜索结果页                      P06 订单页    P08 我的页
                        │
                   点击网点卡片
                        │
                        ↓
                   P03 网点详情页
                        │
                   点击"立即预订"
                        │
                 未登录？→ 触发登录弹窗 → 登录后继续
                        │
                        ↓
                   P04A 搜索路径下单页
                        │
                   确认支付
                        │
                        ↓
                   P05 支付成功页
                        │
                   查看订单详情
                        │
                        ↓
                   P07 订单详情页
```

### 3.1 底部导航 Tab 跳转规则

| Tab | 目标页 | 登录要求 | 未读状态 |
|-----|-------|---------|---------|
| 🏠 首页 | P01 | ❌ 无需登录 | — |
| 📦 订单 | P06 | ✅ 需登录 | 进行中订单数量角标 |
| 👤 我的 | P08 | ✅ 需登录 | — |

**未登录点击"订单"或"我的"Tab**：弹出手机号授权弹窗，授权成功后跳转目标页。

## 四、P01 首页

### 4.1 页面入口条件

| 入口方式 | 条件 | 备注 |
|---------|------|------|
| 冷启动（直接打开小程序）| 无 box_id 参数 | 默认落地页 |
| 运营推广链接 | 无 box_id 参数 | — |
| 分享卡片 | 无 box_id 参数 | — |
| 底部Tab点击 | — | 不刷新，恢复上次滚动位置 |
| ⚠️ 扫码进入 | box_id 参数存在 | **跳过首页，直接到 P04B** |

### 4.2 页面结构与区域定义

```
P01 首页区域划分（从上到下）：

┌───────────────────────────────┐
│  区域A：顶栏                   │  固定，不随滚动消失
├───────────────────────────────┤
│  区域B：Banner 轮播区          │  随滚动上移
├───────────────────────────────┤
│  区域C：搜索区                 │  随滚动上移
├───────────────────────────────┤
│  区域D：附近网点区             │  随滚动上移
├───────────────────────────────┤
│  区域E：金刚区                 │  随滚动上移
├───────────────────────────────┤
│  区域F：活动区                 │  随滚动上移
└───────────────────────────────┘
```

### 4.3 各区域元素状态定义

#### 区域A：顶栏

| 元素 | 默认状态 | 交互行为 | 跳转目标 |
|------|---------|---------|---------|
| 品牌名称"小铁行李寄存丨无忧存" | 静态文字 | 不可点击 | — |
| `[城市▼]` 切换按钮 | 显示当前定位城市 | 点击 → 城市选择底部弹层 | 在当前页更新城市 |

**城市切换弹层行为**：
- 弹出：城市列表（按拼音排序 + 定位城市置顶）
- 选中城市后：关闭弹层，首页附近网点区刷新为新城市数据
- 无法定位时：显示"定位失败，请手动选择城市"

#### 区域B：Banner 轮播区

| 元素 | 状态条件 | 行为 |
|------|---------|------|
| Banner 轮播图 | 运营配置有内容 | 自动轮播（间隔4秒），点击跳运营链接 |
| Banner 轮播图 | 运营配置为空 | 隐藏整个区域B，下方区域上移 |
| 轮播指示点 | — | 跟随轮播联动 |

#### 区域C：搜索区

| 元素 | 默认状态 | 交互行为 | 跳转目标 |
|------|---------|---------|---------|
| 搜索框 | 显示 placeholder "搜索景区 / 地铁站 / 地标" | 点击 → 激活搜索 | 进入搜索输入态（搜索框展开，键盘弹出）|
| 热门标签（最多3个）| 显示运营配置标签 | 点击标签 → 直接搜索该关键词 | P02（带关键词参数）|
| 历史搜索记录 | 有历史搜索时显示（最近3条）| — | — |
| 历史搜索条目 | — | 点击 → 直接搜索 | P02（带关键词参数）|
| 历史搜索"清除" | — | 点击 → 二次确认弹窗 → 确认后清空 localStorage | 留在当前页，区域隐藏 |

**搜索输入态**：
- 输入时实时联想（防抖 300ms）
- 联想结果：地标名称 + 地址（来自后端接口）
- 点击联想结果 → 跳 P02，传入 `keyword` + `poi_id`
- 回车/点击搜索按钮 → 跳 P02，传入 `keyword`
- 点击"取消" → 退出搜索输入态，恢复默认状态

**历史搜索存储规则**：
- 存储位置：localStorage（`wyc_search_history`）
- 格式：`[{keyword, timestamp}]`，按时间倒序
- 上限：保留最近 3 条，超出自动删除最旧记录
- 触发存储时机：用户从 P02 返回首页时（确认搜索有效后写入）
- 无需登录即可使用；小程序卸载后数据丢失，可接受

#### 区域D：附近网点区

| 元素 | 状态条件 | 行为 |
|------|---------|------|
| "📌 我附近有 N 个寄存点" | 定位成功 + 有附近网点 | 动态数量，点击"查看地图→"跳 P02（附近模式）|
| 附近网点小卡片（3张）| 定位成功 + 有附近网点 | 横向排列，点击跳 P03（传 store_id）|
| 卡片内容 | — | 缩略图（不模糊）+ 距离 + 价格起步 |
| 定位失败提示 | 定位失败 | 显示"定位失败，试试搜索附近地标" + 搜索入口 |
| 附近无网点提示 | 定位成功 + 0结果 | 显示"附近 2km 暂无网点，试试扩大范围" + [查看更多→] |

**定位权限处理**：
- 首次进入：请求定位权限
- 用户拒绝：展示"定位失败"态，不再重复弹权限请求
- 已拒绝后：顶部显示"开启定位权限以查看附近网点"引导条

#### 区域E：金刚区

| 元素 | 状态条件 | 行为 | 跳转目标 |
|------|---------|------|---------|
| 📷 扫码存包 | 始终显示（第1位固定）| 点击 → 调起扫码 | 识别 box_id → P04B |
| 🚚 行李配送 | `CITY_OPEN=true` | 点击 → 进入行李配送流程 | 行李配送页（待规划）|
| 🚚 行李配送 | `CITY_OPEN=false` | 点击 → Toast "当前城市暂未开通，敬请期待" | 留在当前页 |
| 🗄️ 长租柜 | 后台配置开启 | 点击 → 长租柜列表页（待规划）| — |
| 更多▼（第4位固定）| 始终显示 | 点击 → 底部半屏弹出全部功能列表 | 弹层内点击各功能 |

**"更多"弹层内容**：
- 显示全部已配置的金刚区功能（含未在主区域显示的）
- 按 `sort_order` 排列
- 无内容时不出现"更多"外的功能项（但"更多"按钮本身始终保留）

#### 区域F：活动区

| 元素 | 状态条件 | 行为 |
|------|---------|------|
| 活动卡片 | 运营配置有活动 | 点击跳运营链接 |
| 空状态兜底 | 运营无配置 | 显示"邀请好友得优惠" + [立即邀请] 按钮 |

## 五、P02 搜索结果页（地图/列表页）

### 5.1 页面入口与参数

| 来源页 | 传入参数 | 页面初始化行为 |
|-------|---------|-------------|
| P01 搜索框搜索 | `keyword`, `poi_id`（可选）| 以关键词定位中心点，加载周边网点 |
| P01 热门标签点击 | `keyword` | 同上 |
| P01 历史搜索点击 | `keyword` | 同上 |
| P01 附近网点"查看地图" | `mode=nearby` | 以用户当前位置为中心，加载附近网点 |
| P01 附近网点卡片点击 | `store_id` | 加载该 store_id 的网点，并在地图和列表中高亮 |

### 5.2 页面视图状态

| 状态 | 条件 | 显示 |
|------|------|------|
| 地图+列表并排 | 默认 | 上半屏地图，下半屏列表可上拖 |
| 纯列表视图 | 用户切换至列表Tab | 全屏列表，地图收起 |
| 地图选中态 | 用户点击地图网点图标 | 该卡片在列表中置顶，地图图标放大高亮 |
| 加载中 | 网络请求中 | 骨架屏（卡片形状占位）|
| 搜索无结果 | 0 条返回 | 空状态页（见§14）|

### 5.3 元素状态定义

#### 顶部搜索栏

| 元素 | 行为 |
|------|------|
| 搜索框（回显搜索词）| 点击 → 返回搜索输入态（与P01搜索输入态一致）|
| `[←]` 返回按钮 | 返回 P01 |
| `[筛选▼]` | 点击 → 筛选面板底部弹层 |

#### 筛选面板

| 筛选类型 | 选项 | 多选规则 |
|---------|------|---------|
| 网点类型 | 人工寄存 / 智能柜 | 单选，可取消 |
| 特色标签 | 可邮寄 / 专人看守 / 有监控 / 可过夜 | 多选 |
| 营业时间 | 现在可用 / 全天 / 自定义时间 | 单选 |
| 价格区间 | 滑动条（最低价~最高价）| — |

筛选项变化后：**实时更新**地图和列表（无需点击确认按钮）

#### 地图区

| 网点类型 | 图标颜色 | 地址显示 |
|---------|---------|---------|
| 人工寄存（`STORE_TYPE=MANUAL`）| 🟡 黄色图标 | 模糊地址（区域级）|
| 智能寄存柜（`STORE_TYPE=LOCKER`）| 🔵 蓝色图标 | 精确地址 |

**地图选中交互**：
1. 点击地图图标 → 图标放大（1.2倍）+ 颜色加深
2. 底部列表：该网点卡片置顶，伴随向上轻弹动效（列表区域上移约 1.5 张卡片高度）
3. 再次点击地图空白区域 → 取消选中，图标恢复，列表回原排序

#### 网点卡片

| 元素 | `STORE_TYPE=MANUAL` | `STORE_TYPE=LOCKER` |
|------|--------------------|--------------------|
| 左侧边框色 | 黄色 | 蓝色 |
| 类型图标 | 🟡 | 🔵 |
| 缩略图 | **不模糊**（缩略图无防爬）| **不模糊** |
| 地址显示 | 模糊地址（如"外滩附近"）| 精确地址 |
| 距离 | 显示（如"0.3km"）| 显示 |
| 价格 | "8元起" | "5元起" |
| 营业状态 | 营业中 / 休息中 | 24h 开放（或具体时间）|
| 标签 | 最多3个（如"可过夜"）| 最多3个 |

**卡片点击行为**：点击卡片任意区域 → 跳 P03（传 `store_id`）

## 六、P03 网点详情页

### 6.1 页面入口与参数

| 来源页 | 传入参数 |
|-------|---------|
| P02 卡片点击 | `store_id` |
| P01 附近网点卡片点击 | `store_id` |

### 6.2 页面状态：信息解锁

本页所有信息按"是否已下单该网点"分两态：

| 状态 | 条件 | 缩写标记 |
|------|------|---------|
| 未下单态 | 用户无该网点的有效订单 | `LOCKED` |
| 已下单态 | 用户有该网点的有效订单（`ORDER_STATUS` 为 PENDING / IN_PROGRESS）| `UNLOCKED` |

### 6.3 各区域元素状态定义

#### 顶部通知条（动态配置）

| 状态条件 | 显示内容 | 行为 |
|---------|---------|------|
| 后台有配置通知内容 | 显示通知条（黄色背景，图标+文字）| 点击可跳运营链接，或仅展示 |
| 后台无配置内容 | **整个通知条隐藏**，不占位 | — |

#### 顶部地图预览区

| 元素 | LOCKED | UNLOCKED | 说明 |
|------|--------|---------|------|
| 地图精确点位 | ✅ 始终可见 | ✅ 始终可见 | 防爬锁的是导航能力，不是位置显示 |
| 导航按钮 | 🔒 灰色，点击弹提示 | ✅ 可点击，拉起地图导航 | — |
| 双指放大 | ✅ 可操作 | ✅ 可操作 | — |
| 上滑进入全屏地图 | ✅ 可操作 | ✅ 可操作 | — |
| 全屏态：下拉/上滑复位 | ✅ 可操作 | ✅ 可操作 | 回到嵌入尺寸 |

**导航按钮 LOCKED 态点击**：弹出底部提示弹层

## 十四、边界场景与异常处理

### 14.1 空状态场景

| 场景 | 文案 | 引导操作 |
|------|------|---------|
| P02 搜索无结果 | "该地标附近暂无寄存点，试试其他地标" | [搜索其他地标] |

| P06 历史订单为空 | "还没有寄存记录" | [去找寄存点→P01] |
| P08 收藏为空 | "还没有收藏的网点" | [去找网点→P01] |

### 14.2 网络异常处理

| 场景 | 处理方式 |
|------|---------|
| 首次加载失败 | 显示骨架屏后转为错误页，含[重新加载]按钮 |
| 下拉刷新失败 | Toast "刷新失败，请检查网络" |
| 支付过程中断网 | 跳支付结果查询页，1分钟后判断支付状态 |
| 搜索联想请求超时 | 取消加载动画，不展示联想结果（不影响正常搜索）|

### 14.3 特殊入口边界

| 场景 | 处理方式 |
|------|---------|
| 扫到无效/已失效二维码 | 弹窗"该寄存点已停止服务，请寻找附近其他网点" + [查看附近→P02] |
| box_id 对应网点已下线 | 同上 |
| 分享链接中的 store_id 不存在 | 跳 P01，Toast "该网点暂时无法访问" |
| 定位权限被永久拒绝 | 顶部引导条：点击可跳系统设置页开启定位 |

### 14.4 取消订单逻辑

```
用户点击"取消订单"
    │
    ├── 判断取消规则
    │       │
    │       ├── 预约开始前 → 提示"确认免费取消？全额退款" → 确认 → 取消成功
    │       │
    │       ├── 预约进行中 → 提示"预约进行中，取消可能扣除部分费用，确认继续？"
    │       │               → 确认 → 进入退款审核流程 → 状态变为 REFUND_PENDING
    │       │
    │       └── 预约已结束 → 提示"订单已结束，无法取消" → 引导联系客服
    │
    └── 取消成功后：订单状态变为 CANCELLED，退款原路退回
```
