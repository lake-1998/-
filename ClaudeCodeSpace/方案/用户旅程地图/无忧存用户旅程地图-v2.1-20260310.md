## 一、文档概述

### 1.1 旅程地图目标

本文档通过系统化梳理无忧存用户从需求产生到后续服务的完整旅程，识别关键痛点和优化机会，为平台合并（小铁寄存柜 + 无忧存）后的UI优化提供决策依据。

### 1.2 旅程阶段划分

用户旅程分为6个核心阶段：

```
需求产生 → 信息获取 → 网点信息查看 → 预订决策 → 线下履约 → 后续服务
(触发)   (首页搜索)  (详情页)      (下单支付)  (到店取包)  (退款复购)
```

### 1.3 分析维度

每个阶段从以下7个维度进行分析：

1. **用户目标** - 该阶段用户想达成什么
2. **关键触点** - 用户接触的页面/功能
3. **用户行为** - 具体操作路径
4. **情绪曲线** - 情绪变化轨迹
5. **核心痛点** - 当前存在的问题（P0/P1/P2分级）
6. **数据指标** - 关键转化率、点击率等
7. **优化机会** - 改进方向和建议

### 1.4 两条完全独立的用户路径（v2.1 核心认知）

> ⚠️ **必须正确理解，影响路由设计**

| 路径 | 入口 | 落地页 | 经过首页？ |
|------|------|-------|----------|
| **扫码流** | URL 含 `box_id` 参数（扫机柜/网点二维码） | 直接跳 **P04B 下单页** | ❌ 不经过首页 |
| **搜索流** | 运营入口/搜索/分享/直接进入 | **P01 首页** → 搜索 → 列表 → 详情 → 下单 | ✅ 经过首页 |

```
路由逻辑（已确认实现方式）：
if (url.params.box_id) {
  redirect → P04B（扫码下单页）
} else {
  render → P01（首页，仅服务搜索流用户）
}
```

> ⚠️ **已废弃逻辑**：v1.0 方案曾设计"扫码进入首页展示特殊布局（场景A/B/C）"以及"智能场景识别动态调整首页"——**此设计已全部废弃，请勿实现**。首页只服务搜索流用户，布局固定。

## 二、用户旅程详细分析

### 阶段1：需求产生（触发阶段）

#### 📋 阶段概览

| 维度 | 内容 |
|------|------|
| **用户目标** | 意识到需要寄存行李，寻找解决方案 |
| **情绪状态** | 😟 焦虑、不确定 |
| **平均时长** | 即时触发 - 数分钟 |

#### 📱 用户场景

1. **规划阶段**：正在规划旅行，查看小红书/马蜂窝攻略时发现需要寄存
2. **临时需求**：已到达目的地，临时发现有寄存需求（如景区不让带包）
3. **空档时间**：退房后到高铁/飞机前的3-5小时空档期

#### 🔗 触达渠道

- **微信搜索**："行李寄存"、"XX景区寄存"（占比60%）
- **小红书/旅游攻略APP**：看到推荐或攻略提及
- **朋友分享**：微信好友/群分享小程序
- **线下扫码**：看到寄存点二维码（占比低，小铁寄存柜为主，扫码直达下单页）

#### 🔴 核心痛点

**P2级痛点**：

1. **品牌认知度低**
   - 问题：微信指数显示"行李寄存"仅90，用户不知道搜什么关键词
   - 影响：流量获取困难，依赖竞价排名
   - 数据：无忧存品牌搜索量 << 存知己、途简单

2. **渠道分散，触达困难**
   - 问题：用户可能在多个平台寻找，不知道从哪里找到服务
   - 影响：需要多渠道投放，成本高

3. **缺乏信任感**
   - 问题：新用户对平台安全性、可靠性存疑
   - 影响：即使找到也不敢下单

#### ✨ 优化机会

- **品牌建设**：提升"无忧存"品牌知名度，SEO优化"行李寄存"关键词
- **内容营销**：小红书/抖音种草，旅游攻略植入
- **信任背书**：首页突出"100万保险"、"XX万用户选择"等信任元素

### 阶段2：信息获取（首页 P01 - 重大痛点区域）🔴

> ⚠️ **本阶段只适用于搜索流用户**。扫码流用户不经过此阶段，直接进入阶段4（下单）。

#### 📋 阶段概览

| 维度 | 内容 |
|------|------|
| **用户目标** | 快速找到适合的寄存网点 |
| **情绪状态** | 🤔😰 比较、困惑、认知焦虑 |
| **平均时长** | 2-5分钟 |
| **流失率** | 估算：30-40%（高获取低转化） |

#### 📱 当前页面结构

```
┌─────────────────────────────────────────┐
│           【顶部三大功能模块】            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 扫码存包  │ │ 行李配送  │ │ 寄存网点  │ │
│  │ (智能柜)  │ │(跳小程序) │ │ 搜索入口  │ │
│  └──────────┘ └──────────┘ └──────────┘ │
│                                          │
│        【寄存网点搜索入口 - 内部】         │
│   ┌────────────────────────────────┐    │
│   │ • 定位 ────┐                   │    │
│   │ • 热门搜索点 ├─→ 地标输入框     │    │
│   │ • 更多 ────┘                   │    │
│   │ • 搜索框 ──┘                   │    │
│   │ • AI搜索推荐（独立入口）        │    │
│   │ • 查找寄存点按钮 → 地图页       │    │
│   └────────────────────────────────┘    │
│                                          │
│        【我的附近寄存点】                 │
│    点击率仅 4.66%                        │

```

> 注：v2.1 已确认首页改动：① 删除品牌Logo导航栏；② 城市选择器内嵌搜索框。三大功能模块和入口冗余问题为下一阶段优化项。

#### 📊 数据指标

| 指标 | 数值 | 说明 |
|------|------|------|
| **首页搜索使用率** | 55%+ | 名称搜索35.63% + 查找按钮19.76% |
| **我的附近点击率** | 4.66% | 使用率极低，与搜索差距13倍 |
| **选择城市** | 15.04% | 用户主动切换城市 |
| **访问来源（无忧存）** | 搜索60% | 线上预约场景为主 |
| **访问来源（小铁柜）** | 扫码80%+ | 线下到店场景为主，**已通过路由分流，不进首页** |

#### 🔴 核心痛点

**P0级痛点（致命问题，必须立即解决）**：

1. **功能优先级混乱，认知负担重** 🔴🔴🔴
   - **问题描述**：
     - 三大功能模块（扫码存包、行李配送、寄存网点）平铺展示，视觉权重相同
     - 搜索流用户带着"我要寄存行李"需求进来，但看到3个选项需要判断
     - "扫码存包"让搜索流用户困惑："我需要先到柜子前吗？"
     - "行李配送"让用户困惑："这是寄存吗？还是快递？"
   - **影响**：
     - 决策疲劳，部分用户直接流失
     - 学习成本高，降低复购效率
     - 首次访问转化率低
   - **数据支撑**：
     - 55%搜索流用户在找网点，说明寄存网点是核心需求
     - 首页"扫码存包"对搜索流用户无实际价值（扫码流已绕过首页）

2. **寄存网点搜索入口内部，入口过多且重复** 🔴🔴🔴
   - **问题描述**：
     - "定位、热门搜索点、更多、搜索框" 四个入口都跳转同一个地标输入框
     - "查找寄存点按钮" 直接跳转地图页
     - 两条路径混在一起，用户不知道选哪个
   - **用户困惑**：
     - "定位"是不是就是"我的附近"？
     - "搜索框"和"热门搜索点"有什么区别？
     - "查找寄存点"和上面那些搜索有什么不同？
   - **影响**：
     - 选择过多导致选择困难
     - 重复功能浪费开发资源
     - 用户路径不清晰，转化率低

**P1级痛点（严重影响体验）**：

3. **AI搜索能力不足** 🔴🔴
   - **现状**：AI功能已上线，但仅3句上下文记录
   - **场景案例**：
     ```
     用户：我明天要去外滩玩，下午3点到，晚上8点走
     AI：推荐XX网点（单次回答）
     用户：有便宜一点的吗？
     AI：（可能记不住"外滩""3点-8点"的上下文）
     ```
   - **影响**：用户无法进行复杂的多轮对话，AI体验打折扣

4. **我的附近模块点击率极低（4.66%）** 🔴🔴
   - **原因分析**：
     1. 位置不佳：放在首页下方，视觉优先级低
     2. 信息不足：只显示简略信息，用户缺乏点击动力
     3. 与搜索重复：用户可能觉得"搜索"就能找到附近的
     4. 命名问题："我的附近"不如"附近XX个寄存点"直观
   - **数据对比**：
     - 搜索相关操作：55%+
     - 我的附近：4.66%
     - **差距13倍！**

5. **网点数量不足，优质网点更少** 🔴🔴
   - **现状**：
     - 商家门店：3353个
     - 覆盖城市：267个
     - 但优质网点（靠近景区、交通枢纽）数量不够
   - **对比竞品**：
     - 存知己：~2000个点位
     - 途简单：1710个点位
   - **影响**：用户搜索后可能找不到合适网点

#### ✨ 优化机会

**高优先级优化（P0）**：

1. **首页统一搜索优先布局**（对应 v2.1 已确认方向）
   - 首页固定为搜索流服务，不再做扫码/搜索场景区分
   - 突出"寄存网点搜索"为核心功能入口
   - 弱化或整合"扫码存包"入口（扫码流已由路由直接处理）
   - 保留"行李配送"但降低视觉权重

2. **合并重复入口，统一搜索路径**
   - **优化方案**：
     ```
     保留：
     - 快速搜索栏（地标名称）→ 地图页/列表页
     - AI搜索（对话式）→ 推荐结果

     删除/合并：
     - 定位（功能并入快速搜索栏）
     - 热门搜索点（改为搜索框下方的快捷标签）
     - 更多（无意义入口）
     ```

**中优先级优化（P1）**：

3. **增强AI对话能力**
   - 上下文记录：3句 → 15句
   - 支持追问："更便宜"、"更近"、"营业到晚上10点"
   - 记忆用户偏好：收藏的网点、常去的城市

4. **重新设计"我的附近"模块**
   - 命名优化："附近X个寄存点"（动态显示数量）
   - 位置上移：放到搜索框下方
   - 增加标签：实时库存、距离、价格起步价
   - 一键导航：直接跳转地图定位

### 阶段3：网点信息查看（详情页 P03 - 致命痛点）🔴🔴🔴

#### 📋 阶段概览

| 维度 | 内容 |
|------|------|
| **用户目标** | 判断这个网点是否符合需求，决定是否下单 |
| **情绪状态** | 😰😠 紧张、疑虑、安全感下降 |
| **平均时长** | 1-3分钟/网点（通常查看3-5个网点对比） |
| **流失率** | 估算：50%+（最大流失点） |

#### 📱 用户核心关注优先级

```
P0级信息（决策关键）：
  1. 距离我的目的地多远？
  2. 具体位置在哪？（地图精确点位可见，精确地址下单后解锁）
  3. 价格多少？
  4. 营业时间是否匹配我的行程？

P1级信息（辅助判断）：
  5. 有没有优惠？
  6. 取消政策如何？
  7. 网点安全吗？评分多少？

P2级信息（次要）：
  8. 收藏功能
  9. 分享功能
  10. 使用手册/操作指引
```

#### 📐 信息解锁策略（v2.1 已确认，影响接口权限）

> ⚠️ **解锁触发点为"下单完成"，与"登录"无关**。详情页不设计"登录后查看"中间态。

| 信息类型 | 解锁时机 | 未解锁展示方式 |
|---------|---------|--------------|
| 精确地址 | **下单完成后** | 区域级描述（如"外滩附近"） |
| 商家电话 | **下单完成后** | 隐藏，改为「下单后解锁」提示区块（⑪ 已删除商家电话模块） |
| 实景大图 | **下单完成后** | 模糊遮罩 |
| 地图精确点位 | **始终显示** | — （锁定的是导航能力，不是点位） |
| 地图导航功能 | **下单完成后** | 导航按钮锁定（🔒图标）（⑥ 已删除未锁状态导航按钮） |

**策略设计意图**：防止用户在未下单前直接飞单（线下现金支付），同时通过"点位可见、地址模糊"保留足够信息让用户做决策。

#### 🔴 核心痛点

**P0级痛点（致命问题）**：

1. **防飞单策略影响下单前体验** 🔴🔴🔴
   - **当前状态**（v2.1 已确认的锁定项）：
     - ❌ 精确地址：下单前不可见，仅显示区域描述
     - ❌ 商家电话：下单前完全隐藏
     - ❌ 实景大图：下单前模糊遮罩
     - ❌ 导航功能：下单前锁定（按钮显示🔒）
     - ✅ 地图精确点位：**始终可见**（可看到在哪，但不能一键导航）

   - **用户心理反应**：
     ```
     用户进入详情页 → 看到地图位置（点位可见）
       ↓
     想确认具体地址 → 显示"外滩附近"，模糊
       ↓
     想联系商家 → 找不到电话
       ↓
     "不放心，安全感不够"
       ↓
     部分用户放弃，转投竞品（存知己、途简单直接显示精确地址）
     ```

   - **竞品对比**：
     - **存知己/途简单**：详情页直接显示精确地址和地图标注，无任何锁定
     - **无忧存（v2.1）**：点位可见，精确地址/电话/导航下单后解锁

   - **业务权衡**：
     - 此策略的核心目标是防飞单（防止绕过平台现金交易）
     - 代价：与竞品相比存在体验落差，详情页转化率偏低
     - 补偿机制（v2.1 新增）：评价模块、寄存保障区块、3步骤流程说明 → 用已知信息建立信任

2. **信息展示无优先级，页面混乱** 🔴🔴🔴
   - **当前问题**：
     - 所有信息平铺展示，没有视觉层级
     - 用户需要的核心信息（距离、价格、营业时间）淹没在其他信息中
     - 边缘功能（分享、使用手册）占据重要位置
   - **v2.1 已改动**（减轻此问题）：
     - ⑦ 价格展示改为双行卡片，可视性更好
     - ⑧ 新增行李寄存流程（3步骤），明确使用路径
     - ⑨ 新增用户评价模块
     - ⑩ 新增寄存保障区块
     - ⑫ CTA栏左侧改为收藏店铺（原价格展示移至卡片）
   - **影响**：
     - 用户需要花时间寻找关键信息
     - 决策效率低

**P1级痛点**：

3. **缺乏对比功能** 🔴🔴
   - **现状**：用户查看网点A → 返回列表 → 查看网点B → 忘记A的信息
   - **影响**：决策困难，需要反复查看
   - **状态**：v2.1 暂未规划此功能

#### ✨ 优化机会

**高优先级优化（P0）**：

1. **防飞单信息策略的体验补偿**（v2.1 已部分实现）

   | 补偿措施 | 实现状态 | 说明 |
   |---------|---------|------|
   | 评价/评分模块 | ✅ v2.1 已新增（⑨） | 用口碑建立信任 |
   | 寄存保障区块 | ✅ v2.1 已新增（⑩） | 100万保险等背书 |
   | 3步骤操作流程 | ✅ v2.1 已新增（⑧） | 降低使用门槛感知 |
   | 下单后解锁预告 | 待规划 | 下单前告知"下单后解锁导航+电话+大图" |
   | 技术防爬（频率限制/设备指纹） | 待规划 | 长期方案，减少对信息锁定的依赖 |
   | 虚拟号码 | 已计划 | 电话解锁后使用虚拟号保护商家 |

2. **重新设计信息架构**（v2.1 已优化，下阶段继续深化）

   **推荐布局**：
   ```
   ┌─────────────────────────────────┐
   │ 【顶部核心信息卡片】             │
   │  📍 距离你的目的地 0.3km        │
   │  💰 8元起/天 (有优惠券)         │
   │  ⏰ 营业中 08:00-22:00          │
   │  ⭐ 4.8分 (238条评价)          │

#### 🔴 核心痛点

**P0级痛点（致命问题）**：

1. **找不到网点（高流失点）** 🔴🔴🔴
   - **问题原因**：
     1. **网点定位不准** - GPS漂移，地图标注错误
     2. **图片只有门头** - 用户到了附近但找不到具体店铺
     3. **电话打不通** - 商家忙/手机没电/不接陌生电话

   - **用户体验路径**：
     ```
     用户下单后获得精确地址 → 导航到附近 → 看不到店 → 打电话 → 没人接
       ↓
     在路上转圈10分钟
       ↓
     焦虑、愤怒 😤
       ↓
     被迫取消（还扣违约金！）💢
       ↓
     差评 + 投诉 + 永不复购
     ```

   - **数据推测**：这个环节可能是**最大的流失点**和**差评来源**

2. **违约金机制不合理（用户视角）** 🔴🔴🔴
   - **当前规则**：
     - 预约日期后取消 → 扣违约金
     - 定位500米内取消 → 扣违约金

   - **不合理场景**：
     ```
     场景1：
     用户按时到达500米内 → 打电话无人接 → 找不到店
     → 被迫取消 → 还要扣违约金！

     场景2：
     用户提前1天取消（计划改变）→ 扣违约金
     但商家实际没有任何损失（还没开始准备）
     ```

   - **对比竞品**：
     - **存知己**：寄存当天才能取消，取消收20%手续费
     - **无忧存**：预约日期后取消就扣费

**P1级痛点**：

3. **飞单问题严重** 🔴🔴⚡
   - **飞单场景**：
     ```
     用户到店 → 商家说"直接给我现金，不用小程序支付"
     用户：可以省手续费吗？
     商家：我给你便宜5元
     用户：好！
     → 平台损失订单
     ```

   - **根本原因**：
     - 平台没有给用户"必须走平台"的理由
     - 商家直收可以省平台分成
     - 用户觉得现金更快

   - **当前防飞单措施不足**：
     - 拍照上传行李（但取消后就无效）
     - 无法要求商家配合确认
     - 无有效惩罚机制

**P2级痛点**：

4. **超时处理需说明** 🔴
   - **当前机制**：超时收取滞纳金，需支付完所有费用才结束订单
   - **问题**：用户可能不清楚具体收费标准

#### ✨ 优化机会

详细的防飞单方案见独立文档《防飞单策略整理-20260306.md》，此处简要总结：

**高优先级优化（P0）**：

1. **增强定位与找店能力**
   - **商家端GPS校准工具**：商家入驻时要求现场校准

   - **AR导航**（技术方案）：类似高德AR导航
   - **紧急联系机制**：商家30秒不接自动转客服

2. **分级违约金 + 特殊豁免**
   ```
   取消时间             违约金    理由
   预约2天前            0%       商家无损失
   预约1天前            10%      商家少量损失
   预约当天-开始前       30%      商家准备成本
   开始后2小时内        50%      已占用资源
   开始2小时后          100%     严重违约

   特殊情况豁免：
   - 商家电话打不通 → 免违约金
   - 定位偏差>50米  → 免违约金
   - 商家提前关店   → 免违约金
   ```

3. **防飞单组合拳（Phase 1）**
   - **阶梯分成**：月订单越多，分成比例越高（70%→85%）
   - **保证金体系**：入驻保证金2000元，违规扣除
   - **保险强绑定**：仅限平台订单享受100万保险
   - **积分会员绑定**：平台订单才计入积分和等级

**中优先级优化（P1）**：

4. **虚拟号 + 自动转接**（已计划）
   - 使用虚拟号码保护商家隐私
   - 商家30秒未接自动转接客服

  // 搜索流：无忧存场景

  // P01首页 → 搜索 → P02地图/列表 → P03详情 → P04A下单
}
```

> ⚠️ **已废弃方案**：v1.0 曾设计"智能场景识别"（扫码→首页特殊布局A/B/C，记住用户偏好），此方案已全部废弃。
