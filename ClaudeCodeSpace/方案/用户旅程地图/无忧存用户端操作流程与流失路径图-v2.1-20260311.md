# 无忧存小程序用户端操作流程与流失路径图（优化前）

**文档版本**: v1.0
**创建日期**: 2026年3月11日
**文档用途**: 基于 UI 优化方案（v2.1）及旅程地图分析，反向还原优化前的用户操作路径与各节点流失情况
**数据来源**: `无忧存运营情况报告.md`、`用户旅程地图梳理-v1.0-20260306.md`、`UI优化方案-20260306.md`（v2.1）
**说明**: 标注 ⚠️ 的数据为文档内的估算值；标注 📊 的数据来自埋点报告实测值

---

## 一、路径全局概览

### 1.1 两条完全独立的用户入口

| 路径 | 触达方式 | 流量占比 | 进入节点 |
|------|---------|---------|---------|
| **路径 A：搜索流** | 微信搜索 / 小程序分享 / 运营推广 | ~60%（无忧存主流量）| 进入首页 |
| **路径 B：扫码流** | 线下扫机柜/网点二维码（含 `box_id` 参数）| ~80%+（小铁寄存柜主流量）| 优化前：进入首页；**优化后：直达下单页** |

> ⚠️ 优化前版本（v1.0 方案）存在设计错误：扫码流被引导进入首页并展示特殊布局，增加了不必要的跳转层级。此逻辑在 v2.0 已废弃。

## 三、路径 B：扫码流（优化前存在设计错误）

```
用户线下扫机柜/网点二维码（URL 含 box_id 参数）
    │
    ▼  ← ⚠️ 优化前错误设计：扫码流被导入首页
┌─────────────────────────────────────────────────────────────────┐
│ 【首页（优化前扫码场景 A/B/C）】  ⚠️ 此逻辑在 v2.0 已废弃       │
│                                                                   │
│  v1.0 错误方案：                                                  │
│   · 场景A：扫码进入，首页突出展示"扫码存包"入口                 │
│   · 场景B：搜索进入，首页突出展示搜索功能                       │
│   · 场景C：首次进入，展示引导（"你是要扫码还是找网点？"）       │
│                                                                   │
│  问题：                                                           │
│  ① 扫码用户已经在柜子/网点面前，不需要再经过首页               │
│  ② 多一个首页跳转 = 多一步流失风险                              │
│  ③ 智能场景识别增加开发复杂度，实际价值有限                     │
└─────────────────────────────────────────────────────────────────┘
    │
    │ 扫码用户额外多走 1 步（首页跳转）
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 【下单页（机柜 or 人工网点）】                                   │
│                                                                   │
│  智能寄存柜：选择柜门大小 → 支付下单                            │
│  人工网点：填写存取时间 + 行李数量 → 支付下单                   │
└─────────────────────────────────────────────────────────────────┘
```

> ✅ **优化后正确路径**：扫码 → **直达下单页**（跳过首页）。扫码流用户不经过首页，首页仅服务搜索流。

## 四、全局流失漏斗（搜索流）

```
用户进入小程序首页
         │  100%
         ▼
   首页功能选择层
         │  ⬇ 流失 30-40%（功能入口混乱，决策疲劳）⚠️
         ▼
   地图/列表找点层
         │  ⬇ 流失 15-20%（无合适网点 / 地图操作不便）⚠️
         ▼
   网点详情页查看层  ← 最大流失点
         │  ⬇ 流失 50%+（地址/电话不可见，违约金措辞，竞品跳失）⚠️
         ▼
   下单确认层
         │  ⬇ 流失 50-60%（优惠券强制/协议抵触）⚠️
         ▼
   支付完成
         │  📊 总转化率约 20.94%（实测）
         ▼
   线下履约
         │  ⬇ 飞单/取消 20-30%（飞单估算）⚠️
         ▼
   有效完成寄存
```

## 五、关键流失节点汇总

| 节点 | 流失率 | 数据性质 | 核心原因 | 优化方向（已在 v2.1 确认）|
|------|-------|---------|---------|------------------------|
| 首页功能入口 | 30-40% | ⚠️ 估算 | 3大入口视觉权重相同，4个搜索入口冗余 | 搜索框置顶，合并入口，删除冗余 |
| 地图/列表找点 | 15-20% | ⚠️ 估算 | 选中地图点位后列表不置顶，人工卡片无图 | 选中置顶动效，卡片补图 |
| 网点详情页 | 50%+ | ⚠️ 估算 | 地址/电话不可见，违约金措辞，实景图模糊 | 区域描述代替模糊地址，退改规则正向表达 |
| 下单页 | 50-60% | ⚠️ 估算 | 优惠券强制，协议无摘要 | 优惠券可取消，总转化率 20.94% 为参考基线 |
| 支付后飞单 | 20-30% | ⚠️ 估算 | 支付成功页无防飞单提醒 | 支付成功页首屏增加防飞单提示 |
| 取消/退款 | 30%+→25% | 📊 实测 | 个人原因取消无约束 | 区分商户/个人原因，个人原因收取违约金 |
