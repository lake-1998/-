## 二、核心业务逻辑（开发必读）

### 2.1 两条用户路径（影响前端路由）

| 路径 | 入口信号 | 落地页 | 说明 |
|------|---------|-------|------|
| 扫码流 | URL 参数含 `box_id` | 直接跳 **P04B 下单页** | 不经过首页，不渲染首页 |
| 搜索流 | 无 `box_id` 参数 | **P01 首页** | 统一搜索优先布局 |

```
路由逻辑（伪代码）：
if (url.params.box_id) {
  redirect → P04B（扫码下单页）
} else {
  render → P01（首页搜索流）
}
```

> ⚠️ v1.0 曾设计"扫码进首页特殊布局（场景A/B/C）"——**此逻辑已废弃**，请勿实现。

### 2.2 信息解锁策略（影响接口权限）

| 信息类型 | 解锁时机 | 未解锁展示 |
|---------|---------|-----------|
| 精确地址 | **下单完成后** | 区域级描述（如"外滩附近"） |
| 商家电话 | **下单完成后** | 隐藏，入口改为「下单后解锁」区块 |
| 实景大图 | **下单完成后** | 模糊遮罩 |
| 地图精确点位 | **始终显示** | — |
| 地图导航功能 | **下单完成后** | 导航按钮锁定（🔒图标） |

> 解锁绑定「下单完成」事件，与「登录」无关。详情页**不需要**设计「登录后查看」中间态。

## 三、各页面改动详情

### P01 首页

#### ① 删除品牌Logo导航栏

| 项目 | 内容 |
|------|------|
| **改动前** | 首页顶部有「小铁行李寄存 \| 无忧存」品牌Logo导航栏，占用44px高度 |
| **改动后** | 删除导航栏，Banner内容上移至状态栏正下方 |
| **改动原因** | 减少首屏无效占屏，扩大可视内容区；合并双品牌后品牌Logo功能弱化 |
| **开发要求** | 移除对应导航组件，首页顶部不再渲染任何品牌标识栏 |
| **验收标准** | 首页打开后，状态栏正下方直接显示Hero Banner，无品牌导航栏 |

#### ② 城市选择器内嵌搜索框

| 项目 | 内容 |
|------|------|
| **改动前** | 城市选择器独立显示于搜索框左侧外部，两个控件分列 |
| **改动后** | `[上海 ▼ \| 🔍 搜索景区/地铁站/地标]` — 城市芯片内嵌搜索框左端 |
| **组件结构** | 搜索框 = 左侧城市Chip（文字+下拉箭头）+ 竖向分隔线（1px #DEDEDE）+ 右侧搜索图标+Placeholder |
| **城市Chip交互** | 点击城市名 → 弹出城市选择面板（已有逻辑，复用） |
| **搜索框交互** | 点击搜索区域 → 进入搜索输入态（已有逻辑，复用） |
| **开发要求** | 搜索框组件重构，将城市选择Chip作为搜索框内联左侧控件 |
| **验收标准** | 搜索框中，城市名与搜索输入区共用一个容器，点击城市和点击搜索互不干扰 |

### P02 地图/列表页

#### ③ 地图Pin区分网点类型

| 项目 | 内容 |
|------|------|
| **改动前** | 所有网点使用统一样式地图Pin |
| **改动后** | 人工网点：橙色圆形Pin（`#FF7043`）+ 「人工·」前缀标签；机柜网点：蓝色圆形Pin（`#2196F3`）+ 「机柜·」前缀标签 |
| **数据依赖** | 后端网点列表接口须返回 `type` 字段：`"manual"` 或 `"locker"` |
| **开发要求** | 地图Pin组件根据 `type` 字段切换颜色和标签文案 |
| **验收标准** | 人工网点Pin为橙色，机柜网点Pin为蓝色，两者在地图上可明确区分 |

#### ④ 网点卡片新增类型标识

| 项目 | 内容 |
|------|------|
| **改动前** | 列表卡片无类型标识 |
| **改动后** | 卡片左侧竖向Badge：人工网点显示「人工寄存」（橙 `#FF7043` 背景）；机柜网点显示「自助机柜」（蓝绿 `#0097A7` 背景），白色文字 |
| **数据依赖** | 同③，依赖 `type` 字段 |
| **开发要求** | 列表卡片左侧Badge组件，根据 `type` 切换文案和背景色 |
| **验收标准** | 人工网点和机柜网点卡片左侧Badge样式明显不同，文案准确 |

#### ⑤ 网点卡片新增营业时间+营业状态

| 项目 | 内容 |
|------|------|
| **改动前** | 卡片无营业时间信息 |
| **改动后** | 卡片中部新增一行：🟢 营业中 · `HH:MM – HH:MM`（或「全天24小时」）；非营业时间显示 🔴 已关闭 |
| **数据依赖** | 后端须返回 `businessHours`（营业时间段）和 `isOpen`（当前营业状态）字段 |
| **展示格式** | `营业状态dot（绿/红）+ 空格 + 「营业中/已关闭」+ 中点 + 营业时间段` |
| **开发要求** | 卡片信息区增加营业时间行；`isOpen` 为 true 显示绿点+「营业中」，false 显示红点+「已关闭」 |
| **验收标准** | 卡片正确展示营业时间和当前营业状态，状态随实际时间变化 |

### P03 网点详情页

#### ⑥ 地图区域导航按钮隐藏

| 项目 | 内容 |
|------|------|
| **改动前** | 地图预览区内有可点击的「导航」按钮 |
| **改动后** | 导航按钮隐藏；地图精确点位**始终显示**（位置锁定的是导航能力，非位置展示）|
| **改动原因** | 防飞单策略：防止用户直接导航绕过平台下单 |
| **解锁方式** | 用户下单后，通过「下单后立即解锁」区块获得导航入口 |
| **开发要求** | 详情页地图组件移除导航按钮；地图本身保持展示，精确点位保持可见 |
| **验收标准** | 详情页地图区域无导航按钮；用户无法在未下单前触发导航 |

#### ⑦ 价格展示重设计（双行卡片）

| 项目 | 内容 |
|------|------|
| **改动前** | 单行价格：「¥5起/件」 |
| **改动后** | 双行价格卡片：第1行「🎒 ¥5/天 · 背包/双肩包」；第2行「🧳 ¥8/天 · 行李箱/拉杆箱」 |
| **数据依赖** | 后端须返回按行李类型区分的价格：`priceBackpack` 和 `priceLuggage` |
| **设计规格** | 两个价格并排卡片，圆角8px，背景 `#FFF9E6`，价格字体加粗，单位/描述文字灰色 |
| **开发要求** | 价格区块重构，从单行文本改为双卡片布局，读取分类价格字段 |
| **验收标准** | 价格区块正确显示背包和行李箱两种定价，格式与设计稿一致 |

#### ⑧ 新增行李寄存流程（3步骤）

| 项目 | 内容 |
|------|------|
| **位置** | 退改规则区块下方 |
| **内容** | 3步流程：① 线上预约 → ② 前往寄存点 → ③ 完成寄存 |
| **展示形式** | 横向步骤条，每步包含图标 + 文字；步骤间有箭头连接 |
| **数据来源** | 静态固定内容（无需接口），全平台统一 |
| **开发要求** | 新增静态「行李寄存流程」区块组件，可复用于所有详情页 |
| **验收标准** | 详情页退改规则下方显示3步流程，图标和文字清晰可辨 |

#### ⑨ 新增用户评价模块

| 项目 | 内容 |
|------|------|
| **位置** | 行李寄存流程下方 |
| **内容** | 评价列表：脱敏手机号（如176\*\*\*\*8088）+ 星级（1-5星）+ 评语文字 |
| **数据依赖** | 后端须提供「网点评价」接口：`GET /store/{id}/reviews?page=1&size=3` |
| **展示数量** | 默认展示最多3条评价，有「查看更多」入口（本期可选） |
| **开发要求** | 新增评价模块组件；手机号前端脱敏处理；星级使用五星图标 |
| **验收标准** | 有评价的网点正确显示评价列表；无评价时隐藏此模块 |

#### ⑩ 新增寄存保障区块

| 项目 | 内容 |
|------|------|
| **位置** | 用户评价下方（页面倒数第2区块，在「下单后解锁」之上） |
| **内容** | 4项保障：官方认证 / 交易保障 / 安全险 / 视频监控 |
| **展示形式** | 2×2图标网格，每项含图标 + 文字标签 |
| **数据来源** | 静态固定内容，全平台统一 |
| **开发要求** | 新增静态「寄存保障」区块组件，可复用 |
| **验收标准** | 所有详情页底部显示4项保障，样式与设计稿一致 |

#### ⑪ 隐藏商家电话模块

| 项目 | 内容 |
|------|------|
| **改动前** | 营业时间区块右侧或下方展示商家电话，用户可直接拨打 |
| **改动后** | 电话信息隐藏，不在详情页展示 |
| **改动原因** | 防飞单策略：防止用户直接联系商家绕过平台下单 |
| **解锁方式** | 用户下单后，在订单详情页或「下单后解锁」区块获得商家电话入口 |
| **开发要求** | 详情页移除电话展示组件；下单成功后在订单详情页显示商家联系方式 |
| **验收标准** | 详情页未下单时看不到任何电话信息；下单后可在订单页联系商家 |

#### ⑫ CTA栏左侧改为收藏店铺

| 项目 | 内容 |
|------|------|
| **改动前** | CTA栏左侧显示「¥5起/件」价格信息 |
| **改动后** | CTA栏左侧改为「♡ 收藏店铺」（心形图标 + 文字，未收藏时灰色 `#CCCCCC`，已收藏时橙色 `#FF8000`）|
| **数据依赖** | 后端须提供收藏接口：`POST /store/{id}/favorite`（收藏/取消收藏）和状态查询 |
| **交互** | 点击心形图标 → 切换收藏/取消状态；未登录时点击 → 弹出登录引导 |
| **开发要求** | CTA左侧组件替换为收藏按钮；需维护收藏状态（本地+服务端同步） |
| **验收标准** | 收藏按钮状态正确（已收藏/未收藏）；收藏操作实时反馈；刷新后状态保持 |

### P04A 下单页

#### ⑬ 删除备注输入行

| 项目 | 内容 |
|------|------|
| **改动前** | 下单表单中包含「备注」文本输入框（非必填）|
| **改动后** | 移除备注输入行 |
| **改动原因** | 备注字段使用率极低，增加用户操作负担；下单流程应尽可能精简 |
| **开发要求** | 下单页表单移除备注字段；提交接口中备注参数可随之废弃或保留默认空值 |
| **验收标准** | 下单页面无备注输入框；下单提交流程中无备注相关步骤 |

## 四、接口需求汇总

| 接口 | 方法 | 改动类型 | 说明 |
|------|------|---------|------|
| 网点列表 | `GET /stores` | 🟠 修改 | 响应须新增 `type`（`manual`/`locker`）、`businessHours`、`isOpen` 字段 |
| 网点详情 | `GET /stores/{id}` | 🟠 修改 | 响应须新增 `priceBackpack`、`priceLuggage`、`type`、`isOpen`、`businessHours` 字段 |
| 网点评价 | `GET /stores/{id}/reviews` | 🟢 新增 | 分页返回评价列表，字段：`maskedPhone`、`rating`（1-5）、`content`、`createdAt` |
| 收藏网点 | `POST /stores/{id}/favorite` | 🟢 新增 | Body：`{action: "add"\|"remove"}`；返回最新收藏状态 |
| 收藏状态查询 | `GET /stores/{id}/favorite` | 🟢 新增 | 返回 `{isFavorited: boolean}` |
| 下单提交 | `POST /orders` | 🟠 修改（可选）| 可移除 `remark` 字段，或保留兼容 |
