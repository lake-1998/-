# 无忧存用户端 · UI 改动需求文档

**文档版本**：v1.1
**整理日期**：2026-03-10
**适用端**：微信小程序 / 支付宝小程序 / 抖音小程序 / H5
**状态**：待产品审批 → 交付设计/研发

> **v1.1 更新说明**：v1.0 基于设计方案编写，存在大量与实装版本重复的内容。v1.1 对照实际运行中 UI（版本合并 v1.0-20260309 截图）全面重新梳理，**剔除已实装功能，只保留真正需要改动的项**。

---

## 一、项目背景与优化目标

### 1.1 背景

小铁寄存柜与无忧存平台合并，两套用户端合并为一套统一 C 端产品（当前实装版本为版本合并 v1.0）。
本次改动基于对实装 UI 的逐页审查，聚焦**防飞单强化、信息架构优化、转化链路提升**三个方向的未完成项。

**关键数据基线**（来源：运营数据报告）：

| 指标 | 当前值 | 优化目标 |
|------|-------|---------|
| 首页搜索使用率 | ~55%（名称搜索 35.6% + 查找按钮 19.8%）| ≥65% |
| 附近网点点击率 | 4.66% | ≥15% |
| 估算飞单率 | 20-30% | 降低 |

### 1.2 两条核心用户路径

| 路径 | 入口信号 | 落地页 | 是否经过首页 |
|------|---------|-------|-----------|
| **扫码流** | URL 参数含 `box_id` | 直接跳 **扫码下单页** | ❌ 不经过首页 |
| **搜索流** | 无 `box_id` 参数 | **首页** | ✅ 经过首页 |

### 1.3 信息解锁策略（当前实装状态 vs 目标状态）

| 信息类型 | 当前实装状态 | 目标状态 |
|---------|-----------|---------|
| 地图精确点位 | ✅ 始终可见 | ✅ 始终可见（不变）|
| 导航按钮 | ⚠️ 按钮可见，点击弹锁定提示（假锁）| 🔒 按钮彻底隐藏，改为解锁引导区块 |
| 精确地址文字 | 🔒 显示区域级描述 + 红字提示 | 🔒 改为视觉更强的解锁引导卡片 |
| 商家电话 | ⚠️ 按钮可见，点击弹锁定提示（假锁）| 🔒 按钮彻底隐藏，下单后在支付成功页展示 |
| 实景大图 | 暂不处理 | 暂不处理 |

## 二、各页面改动需求

> **说明**：以下仅列出当前实装版本中**缺失或需变更**的项。
> 已在 v1.0 实装版本中完成的功能（地图 Pin 区分、卡片类型 Badge、营业时间显示、双价格卡片、用户评价模块、行李寄存流程、寄存保障 4 项、收藏按钮功能、底部 3 Tab、订单列表双 Tab 等）**不再重复列出**。

---

### 首页

#### 改动 A：删除双品牌 Logo 导航栏

| 项目 | 内容 |
|------|------|
| **当前状态** | 首页顶部有「小铁 \| 无忧存」黄色 Logo 导航栏，占用约 44px 高度 |
| **目标状态** | 删除，Banner 直接从状态栏正下方开始 |
| **改动原因** | 减少首屏无效占屏；版本合并完成后双 Logo 展示已无必要 |
| **开发要求** | 移除 Logo 导航组件；首页不渲染任何品牌导航栏 |
| **验收标准** | 首页打开后，状态栏正下方直接显示 Banner，无品牌 Logo 栏 |

#### 改动 B：搜索区显示历史记录 + 热门标签

| 项目 | 内容 |
|------|------|
| **当前状态** | 首页搜索框下方无历史记录或热门标签展示 |
| **目标状态** | 搜索框区域展示：**热门标签**（最多 3 个，运营配置）+ **历史搜索记录**（最近 3 条） |
| **交互规则** | 点击热门标签或历史记录 → **直接跳地图列表页**（不经过「选择地标」子页） |
| **存储规则** | 历史搜索存 `localStorage`（key: `wyc_search_history`），上限 3 条，按时间倒序 |
| **写入时机** | 从地图列表页返回首页时写入（确认有效搜索后），无需登录 |
| **验收标准** | 有历史记录时首页展示；点击标签/记录直接进入地图列表，不弹搜索子页 |

#### 改动 B2：新增 AI 智能搜索入口

| 项目 | 内容 |
|------|------|
| **当前状态** | 首页无 AI 辅助搜索能力，用户只能通过地标关键词搜索 |
| **目标状态** | 搜索框旁新增 AI 聊天入口（图标或"AI 帮我找"文字按钮），支持用户用自然语言描述需求，由 AI 推荐合适网点 |
| **示例场景** | 用户输入"明天上午我在南京东路游玩，帮我找个附近便宜的寄存点" → AI 返回推荐列表并说明原因 |
| **交互方式** | 点击入口 → 打开 AI 对话**独立页** → 对话结束后返回网点**列表页**（展示 AI 推荐的网点结果）|
| **推荐能力** | 根据用户描述的位置、时间、行李类型、价格偏好进行综合推荐；可多轮对话澄清需求 |
| **降级策略** | AI 无法理解时，引导用户使用普通搜索 |
| **开发要求** | 待确认 AI 能力方案（见待确认事项 #1）|
| **验收标准** | 首页有 AI 搜索入口；自然语言描述后能返回有效的网点推荐结果 |

#### 改动 M：附近网点区规范

| 项目 | 内容 |
|------|------|
| **当前状态** | 首页有「附近寄存点」区块，展示 3 张横排小卡片；卡片信息和状态处理规则不完整 |
| **目标状态** | 区块头部：`📍 附近 N 个寄存点` + `[查看全部 →]`（N 动态更新，点击跳地图/列表页）|
| **卡片内容** | 网点缩略图（不模糊）+ 类型 Badge（人工/机柜）+ 网点名称 + 🟢/🔴 营业状态（同改动 D）+ 距离 + 价格起步 |
| **定位成功·有网点** | 展示最近 3 个网点横排卡片；点击卡片跳对应网点详情页 |
| **定位成功·无网点** | 隐藏卡片区，显示文案「附近 2km 暂无网点，换个地方试试」+ [搜索地标 →] 入口 |
| **定位失败** | 隐藏卡片区，显示文案「定位失败，试试搜索地标」+ [搜索地标 →] 入口 |
| **验收标准** | 三种状态展示正确；营业状态圆点实时准确；点击卡片正确跳转 |

#### 改动 N：金刚区规范与调整

| 项目 | 内容 |
|------|------|
| **区域规则** | 单行 **4 坑位**，第 4 位**固定**为「更多 ▼」，不被其他功能挤占 |
| **坑位分配** | 位置 1：📷 扫码存包（固定第 1 位，全城市）；位置 2-3：由后台配置；位置 4：▼ 更多（固定）|
| **当前功能项** | 扫码存包 / 行李配送 / 长租柜（三项，后两项顺序后台可配置）|
| **行李配送** | 仅限城市白名单开放（当前：福州 / 厦门地铁 / 杭州地铁）；未开通城市点击后 Toast 提示「当前城市暂未开通，敬请期待」|
| **标签规范** | 每个功能项标签 ≤ 4 字（如「扫码存包」而非「扫码」）|
| **后台配置** | 位置 2-3 的功能项及顺序支持后台配置，无需发版；图标和标签同步可配 |
| **验收标准** | 第 4 位始终为「更多」；行李配送在非白名单城市点击有提示不跳转；修改后台配置后前端展示同步更新 |

### 选择地标页（点击搜索框进入的子页）

#### 改动 C：增加历史记录 + 热门标签，支持直接跳转

| 项目 | 内容 |
|------|------|
| **当前状态** | 页面仅有搜索框 + 景点 / 商圈 / 机场车站 / 地铁站分类标签 |
| **目标状态** | 搜索框下方增加：**热门标签**（全量展示，运营配置）+ **历史搜索记录**（最近 3 条，可清除） |
| **交互规则** | 点击热门标签 / 历史记录 → **直接跳地图列表页**（等同手动输入后搜索） |
| **与首页的关系** | 首页展示部分（最多 3 个标签、3 条记录），本页展示全量；数据同源（共用 `localStorage`） |
| **清除历史** | 提供"清除"入口；点击 → 二次确认弹窗 → 确认后清空 |
| **验收标准** | 标签和历史记录点击后直接跳转，无需手动输入；清除后两处均消失 |

### 地图 / 列表页

#### 改动 D：卡片增加当前营业状态指示

| 项目 | 内容 |
|------|------|
| **当前状态** | 卡片仅显示营业时间文字（如 `00:00-24:00`），无当前是否营业的状态标识 |
| **目标状态** | 时间前增加状态圆点：🟢 **营业中** · `00:00-24:00`；或 🔴 **暂停营业** · `09:00 重新开放` |
| **数据依赖** | 后端返回 `isOpen: boolean` 字段（见待确认事项 #1） |
| **展示位置** | 列表卡片营业时间行（地图下方卡片 + 纯列表卡片均需显示）|
| **验收标准** | 状态实时准确；营业中绿点，非营业时间红点；显示下次开放时间 |

#### 改动 O：地图与网点卡片交互规范

**地图区域规范**

| 元素 | 规范 |
|------|------|
| 人工寄存 Pin | 圆形橙色（`#FF7043`），含网点图标 |
| 智能机柜 Pin | 方形黄色（`#FFB300`），含机柜图标 |
| 当前定位 | 蓝色圆点，实时更新 |
| 地图右侧控件 | 定位回中按钮（回到当前位置）/ 客服入口按钮 |
| 地图悬浮标 | 「🛡 寄存安全保障」悬浮徽章（当前已有，保持）|
| Pin 选中态 | 点击 Pin → Pin 放大 1.2 倍；底部卡片列表滚动至该网点置顶 |
| 取消选中 | 点击地图空白区域 → Pin 恢复原尺寸，列表回默认排序 |

**网点卡片规范**

| 元素 | 内容与规则 |
|------|-----------|
| 缩略图 | 左侧竖向图片；人工寄存 + 智能机柜图片均**清晰显示**，不做模糊处理（卡片尺寸小，爬虫价值低，模糊影响第一印象）|
| 类型 Badge | 覆盖在图片左上角：「人工寄存」橙色（`#FF7043`）/ 「自助寄存」青色（`#0097A7`）|
| 网点名称 | 字体加粗，最多 2 行，超出截断 |
| 服务标签 | 可过夜 / 可邮寄 / 专人看守 / 有监控等，最多展示 3 个，超出省略 |
| 营业状态 | 🟢 营业中 / 🔴 暂停营业（同改动 D，与详情页保持一致）|
| 距离 | 「距当前位置直线距离 Xm / Xkm」，动态计算 |
| 价格 | 「¥X / 件 · 天 起」，取背包和行李箱中较低价 |
| 评分 | 星级评分数字（如 5.0）|
| 操作按钮 | 右侧「查看」按钮，点击跳网点详情页；点击卡片内容区同效果 |

**无网点 / 空状态**

| 状态 | 展示 |
|------|------|
| 附近有网点 | 默认排序：综合推荐（距离 + 评分）|
| 附近 2km 无网点 | 全屏地图 + 文案「附近 2km 内暂无寄存点」 + [这里需要寄存点] 按钮 |
| 已提交反馈 | [这里需要寄存点] 按钮变为文案「😊 感谢反馈~ 正在努力开拓中」，不可重复点击 |

**地图 / 列表切换**

- 页面支持地图 + 列表同屏（当前实装模式）和仅地图全屏模式（P03E）
- 向上滑动卡片列表 → 地图区域缩小，列表展开；向下滑动 → 地图恢复

### 网点详情页

#### 改动 F：彻底隐藏「导航」「电话」按钮

| 项目 | 内容 |
|------|------|
| **当前状态** | 地图右上角有「电话」「导航」两个可见按钮；点击后弹"预定后可查看位置或电话联系商家" |
| **目标状态** | 两个按钮**完全移除**；地图区不渲染任何导航 / 电话交互入口 |
| **改动原因** | 按钮可见会吸引用户反复尝试绕过平台；当前"假锁"模式防飞单效果弱 |
| **地图精确点位** | 仍保持始终可见（只锁导航能力，不隐藏位置）|
| **开发要求** | 地图组件移除右上角 Action 按钮渲染 |
| **验收标准** | 详情页地图区无导航、电话按钮；地图点位仍可见 |

#### 改动 G：强化「下单后解锁」引导区块

| 项目 | 内容 |
|------|------|
| **当前状态** | 地址行下方有红色小字"详细地址及商家电话下单后可查看"，样式弱、易被忽略 |
| **目标状态** | 改为独立的**解锁引导卡片**，传递下单后的获益感 |
| **建议样式** | 浅黄色底卡片（`#FFF9E6`）；内容：`🔒 下单后立即解锁` + 三行：`📍 精确地址` / `📞 商家电话` / `🗺️ 一键导航`；右侧附 `[立即预订 →]` 文字链 |
| **位置** | 营业时间 / 地址行下方，价格区块上方 |
| **验收标准** | 卡片样式醒目；点击"立即预订"直接触发下单流程；下单后该区块消失 |

#### 改动 H：营业状态显示（同改动 D）

详情页营业时间行同步显示 🟢 / 🔴 状态圆点，与列表卡片保持一致。

#### 改动 I：CTA 栏左侧改为「收藏」按钮

| 项目 | 内容 |
|------|------|
| **当前状态** | 底部 CTA 栏：左侧「分享店铺 \| 使用帮助」，右侧「⚡ 立即预订」；收藏按钮在页面 Header 右侧（★ 图标）|
| **目标状态** | CTA 栏**左侧**改为「♡ 收藏」（心形图标 + 文字）；Header 原位置的收藏图标移除 |
| **状态样式** | 未收藏：灰色 `#CCCCCC`；已收藏：橙色 `#FF8000` |
| **交互** | 点击切换收藏状态；未登录点击 → 弹出登录引导弹窗 |
| **接口** | `POST /stores/{id}/favorite`（body: `{action: "add"\|"remove"}`）；`GET /stores/{id}/favorite` |
| **验收标准** | CTA 栏左侧为收藏按钮；Header 无重复入口；收藏状态刷新后保持 |

### 支付成功页

#### 改动 K：新增解锁信息卡（当前版本核心缺失项）

| 项目 | 内容 |
|------|------|
| **当前状态** | 支付成功页仅显示"预定成功"图标 + 订单时间说明 + [查看订单] 按钮 + "点击联系商家"文字链；精确地址和导航均未展示 |
| **目标状态** | "预定成功"文字下方新增**解锁信息卡**，展示刚解锁的信息 |
| **卡片内容** | `📍 精确地址（文字）` + `[一键导航]` 按钮；`📞 商家电话（号码）` + `[拨打电话]` 按钮 |
| **改动原因** | 支付后是用户最需要导航 / 联系商家的时刻；当前版本在这个最佳节点没有展示解锁信息，浪费了防飞单最后一道正向引导机会 |
| **数据来源** | 支付成功接口返回 `storeAddress`、`storePhone`（或复用详情接口传 `order_id`）|
| **验收标准** | 支付成功页显示精确地址和商家电话；导航和拨打功能正常 |

## 四、优先级排期

### P0：立即启动（1 个月内）

| 改动 | 页面 | 说明 |
|------|------|------|
| A. 删除双品牌 Logo 导航栏 | 首页 | 纯前端，低风险 |
